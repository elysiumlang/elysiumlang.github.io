CLI Compiler

The elysium (or ely) CLI is the primary interface for compiling and running Elysium programs. It's a Rust compiler with an LLVM backend.

Commands

CommandDescription
elysium build <file>Compile an .ely (plain) or .elyx (UI component / XML-syntax) file to a native binary
elysium run <file>Compile and run immediately
elysium check <file>Type-check without code generation
elysium test [path]Run spec-driven tests (scans tests/ or given file)
elysium update [package]Check EPM registry for newer dependency versions
elysium migrate [path]Auto-migrate source code to latest syntax
elysium replStart an interactive REPL session
elysium doc <file>Generate documentation from /// Summary: comments
elysium deps <file>Show the dependency graph of a program
elysium env <name>Set the active environment (local/dev/test/prod)
elysium highlight <file> --format <ansi|html>Output syntax-highlighted source
elysium port <file.js>Port JavaScript/TypeScript to Elysium

Environment Flag

Use --env (or --env-file) to control which environment-specific stubs are included:

elysium build app.ely ---env test
elysium run app.ely ---env dev
elysium check app.ely ---env prod

Test Options

--dry-run lists specs and feats without type-checking. Default env for tests is test.

Update Options

FlagDescription
--apply / -aWrite updated version constraints to elysium.json
--latestUpgrade to absolute latest version (ignores constraint range)
--forceAllow downgrade if latest is lower than current

Migrate Options

FlagDescription
--checkExit non-zero if any file needs migration (CI mode)
--dry-runShow what would change without writing
--forceApply migrations marked as "requires manual review"

Registered migrations: webworker-to-worker, bm-to-bench, normalize-imports, describe-to-spec.

Build Pipeline

  1. Source → Lexer — Tokenization via logos
  2. Parser — Recursive-descent parsing to AST
  3. Desugaring — Import aliases, builtin calls (console.*, fs.*, string.*, etc.)
  4. Environment Stub Filtering — Remove stubs not matching --env
  5. Type Checker — Full type inference and checking
  6. Ownership Checker — Validate only let semantics
  7. HIR & MIR — High and Mid-level IR lowering
  8. LLVM IR Codegen — Generate native code via inkwell
  9. Native Binary — Output executable (or WASM for browser target: "wasm")

npm Package

Published as elysium-lang on npm. Ships the full Rust compiler and a complete JavaScript runtime.

Installation

npm install -g elysium-lang

# Now you can use the CLI
elysium run hello.ely
ely --version

JavaScript Runtime

Use Elysium's standard library from JavaScript:

const elysium = require('elysium-lang')
elysium.console.log('Hello from Elysium runtime!')
elysium.string.sha256('hello')
elysium.datetime.now()

EPM — Elysium Package Manager

EPM manages dependencies for Elysium projects using flat dependency resolution. It ships as a Rust crate.

Commands

CommandDescription
epm init [name]Scaffold a new package (creates elysium.json + main.ely)
epm install [package]Install all deps or a specific package
epm install --shakeInstall and tree-shake in one step
epm install --legacyAllow multiple versions of the same dependency
epm lockGenerate elysium.lock from current resolution
epm publishPublish @org/package to the registry (GitHub sign-in required)
epm loginSign in with GitHub via gh (EPM stores no secrets)
epm whoamiShow your GitHub username
epm org create <org>Create an org scope you own
epm org listList orgs you own
epm grant <user>Grant publish access (package owner only)
epm search <query>Search registry by name or description
epm info <package>Show package details and versions
epm treeShow the flat dependency tree
epm why <package>Trace how a package is pulled in
epm shake [--dry-run]Tree-shake installed packages
epm listList installed packages in elysium_modules/

Publishing packages

EPM is separate from the compiler. Publishing uses your GitHub account (no EPM signup, no PAT file in ~/.epm). Install the GitHub CLI, then:

epm login
epm org create myteam
epm init --org myteam my-lib
epm publish

Package names must be scoped: @org/name. Only the org owner can publish new packages under that org. The publisher becomes the package owner. Only the owner can run epm grant <github-user> to let someone else publish updates.

Flat Dependency Resolution

By default, EPM uses flat resolution — one version per package name. The algorithm walks all transitive dependencies, finds the highest semver version satisfying all constraints, and pins it in elysium.lock.

Use --legacy to allow multiple versions (first-come-first-served).

Project Manifest (elysium.json)

The elysium.json at the project root defines the project metadata, dependencies, and build targets.

{
    "name": "my-app",
    "version": "1.0.0",
    "description": "My Elysium app",
    "entry": "main.ely",
    "license": "MIT",
    "dependencies": {
        "http": "^1.0.0",
        "json": ">=2.0.0"
    },
    "environments": {
        "local": "local",
        "dev": "dev",
        "test": "test",
        "prod": "prod"
    },
    "ui": {
        "browser": {
            "target": "js"   // "js" or "wasm" (WebAssembly for performance)
        }
    },
    "ssr": {
        "enabled": false,
        "runtime": "node"  // "node", "deno", "bun", "elysium-server"
    }
}

UI Config: ui.browser.target controls the browser compilation target. Set to "wasm" for WebAssembly (performance mode for heavy UI components). Future targets include android and ios.

SSR Config: When ssr.enabled is true, the compiler produces server-rendered output for the specified runtime. Use this when your code targets a Node.js/Deno/Bun server environment.

Linter

The built-in linter checks for common issues and enforces documentation standards:

  • Missing /// Summary: comments on public functions and types
  • Unused variables and imports
  • Unreachable code
  • Invalid bc assertion patterns
  • Incorrect environment stub configurations

Syntax Highlighting

The compiler can output syntax-highlighted source in ANSI (terminal) or HTML format.

# Highlight for terminal
elysium highlight app.ely --format ansi

# Highlight as HTML (embed in docs)
elysium highlight app.ely --format html > app.html

Documentation Generation

Generate documentation from enforced /// Summary: doc comments.

elysium doc app.ely    # Generate documentation
elysium deps app.ely   # Show dependency graph

Elysium also supports a TypeScript/JavaScript porting tool:

elysium port app.js    # Port JS/TS to Elysium

Environment Filtering

Stub functions can be restricted to specific environments. This is useful for mocking during development without affecting production builds.

// Only available during local development
func mockDatabase() -> Data stub: [local, dev]

// Available everywhere
func genericStub() -> String stub

Supported built-in environments: local (default), dev, test, prod. Custom aliases can be defined in elysium.json.
