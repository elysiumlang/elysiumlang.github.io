1. Variables & Mutability

Variables are declared with let (immutable) or var (mutable). Use only let for exclusive ownership.

KeywordSemantics
letImmutable binding
varMutable binding
only letExclusive ownership (compile-time checked)
let name = "Alice"        // immutable
var count = 0             // mutable
count = count + 1

only let data = readFile("data.txt")   // exclusive ownership

2. Strings & Literals

Standard Strings

Double-quoted strings with escape sequences (\n, \t, \", etc.).

let name = "Alice"
let escaped = "Line1\nLine2"
let quoted = "She said \"hello\""

Backtick Strings (Multi-line)

Backtick-delimited strings support multi-line content and double quotes without escaping. Perfect for JSON, HTML, or any text with embedded " characters.

// Multi-line with embedded quotes
let json = `{
    "name": "Elysium",
    "version": "2.0"
}`

// Single line with double quotes
let html = `<div class="container">Hello</div>`

Inline JSON / Record Literals

Create object literals directly with { "key": value, ... } syntax. Desugars to __json_buildObject() at compile time.

// Inline JSON record
let body = {
    "model": "gpt-4o",
    "temperature": "0.7",
    "max_tokens": "100"
}

// Combined with backtick strings for cleaner code
let payload = { "prompt": `Tell me a story` }

is Operator — Runtime Type Check

The is operator checks if a value is an instance of a given type or class at runtime:

class Animal {}
class Dog extends Animal {}
class Cat extends Animal {}

let pet = Dog()
let isDog = pet is Dog      // true
let isAnimal = pet is Animal  // true (subclass)
let isCat = pet is Cat       // false

// With primitive types
let val = "hello"
let isStr = val is String   // true
let isInt = val is Int      // false

3. Functions

Basic Functions

/// Summary: Add two integers.
func add(a: Int, b: Int) -> Int {
    a + b
}

// Single-expression body (no braces needed)
func square(x: Int) -> Int { x * x }

Lambdas

let square = (x) -> x * x
let square = { x -> x * x }

Rest Parameters

func average(…values: Float) -> Float {
    sum(values) / values.count
}

Generic Functions

func identity<T>(x: T) -> T { x }

func first<T>(arr: [T]) -> T? {
    if arr.isEmpty { nil }
    else { arr[0] }
}

Private & Lazy Modifiers

private func helper() -> Int {
    return 42
}

lazy func expensiveComputation() -> Int {
    return helper() * 2
}

private lazy func cachedResult() -> String {
    return "computed-lazy"
}

Stub Functions

Stub functions declare an interface without a body. They can be filtered by environment at compile time.

/// A stub available in all environments
func placeholder(x: Int) -> Int stub

/// Only exists in local and dev environments
func debugOnly(key: String) -> String stub: [local, dev]

4. Control Flow

if / else

// Statement form
if age >= 18 {
    print("Adult")
} else {
    print("Minor")
}

// Expression form (returns a value)
let status = if age >= 18 then "adult" else "minor"

if … then Expression

The then keyword is mandatory in expression form, avoiding ambiguity with statement if.

let status = if age >= 18 then "adult" else "minor"
let discount = if total > 100 then 0.1 else 0.0

for Loops

for item in [1, 2, 3, 4] {
    print(item)
}

for i in 1…5 {        // inclusive range: 1,2,3,4,5
    print(i)
}

for i in 1..5 {        // exclusive range: 1,2,3,4
    print(i)
}

while Loops

while condition {
    // body
}

match (Pattern Matching)

match value {
    case pattern => expression
    case pattern => expression
}

// Example with Result
let result = parseNumber("42")
match result {
    case ok(num) => print(num)
    case err(msg) => print("Error: " + msg)
}

switch / case / then / else

A human-friendly alias for match that reads like natural language.

func describe_score(score: Int) -> String {
    return switch score {
        case 100 then { "Perfect!" }
        case 90 then { "Excellent!" }
        case 80 then { "Great!" }
        case 70 then { "Good" }
        case 60 then { "Passing" }
        else { "Needs improvement" }
    }
}

6. Classes

Classes support fields, methods, constructors (init), and access control with private.

class Car {
    let make
    let model

    init(make, model) {
        this.make = make
        this.model = model
    }

    func drive() {
        print("Driving the ", this.make, " ", this.model)
    }
}

Private Fields & Methods

class User {
    let name: String
    private let email: String

    init(name: String, email: String) {
        // ...
    }

    private func validate() -> Bool {
        return true
    }
}

7. Error Handling

Try / Catch / Finally

try {
    performRiskyOperation()
} catch ErrorType.Network {
    print("Network error occurred.")
} finally {
    cleanup()
}

Result Type

func parseNumber(s: String) -> Result<Int, String> {
    if let num = Int(s) {
        Result.ok(num)
    } else {
        Result.err("invalid number: " + s)
    }
}

// Error propagation with ?
let value = doRisky()?   // returns early if Result is err

8. Memory Model

Elysium uses Automatic Reference Counting (ARC) — no garbage collector pauses, deterministic deallocation.

  • ARC — Automatic Reference Counting (like Swift / Val)
  • weak — Weak reference (does not increase retain count, auto-nilled)
  • unowned — Unowned reference (does not increase retain count, assumes lifetime)
  • only let — Exclusive ownership, compile-time checked, no aliasing
  • unsafe { } — Opt-in block for raw pointers (use sparingly)

9. Concurrency

Async / Await

// Async function
async func fetchData() -> String {
    let data = await httpGet("https://example.com")
    return data
}

// Lightweight tasks
async { producer(ch) }
async { consumer(ch) }

Channels

let ch = Channel<Int>(capacity: 3)

async func producer(ch: Channel<Int>) {
    for i in 1..5 {
        await ch.send(i)
    }
    ch.close()
}

async func consumer(ch: Channel<Int>) {
    while let value = await ch.receive() {
        print("Got: ", value)
    }
}

Parallel Blocks

Each statement inside a parallel block runs in its own thread (using pthreads in the C backend, or Promise.all in the JS runtime).

func main() {
    parallel {
        print("task a\n")
        print("task b\n")
        print("task c\n")
    }
    print("all parallel tasks done\n")
}

Select on Channels

select {
    case msg = ch1 => handle(msg)
    case msg = ch2 => handle(msg)
}

9. Schedule & Wait

Schedule (Cron / Friendly Intervals)

Run functions on a timer using schedule "expr" func name() { }. Supports standard cron expressions and friendly formats:

// Friendly interval
schedule "every 10 minutes" func tick() {
    print("tick\n")
}

// Time of day
schedule "daily at 08:00" func morningRoutine() {
    print("good morning\n")
}

// Cron-style
schedule "*/5 * * * *" func cronJob() {
    print("every 5 minutes\n")
}

// Day of week
schedule "every Monday at 09:00" func weeklyStandup() {
    print("standup time!\n")
}

// Schedule can be combined with async
schedule "every hour" async func healthCheck() {
    let status = await httpGet("https://example.com/health")
    console.log(status)
}

Supported cron aliases: hourly, minutely, daily, weekly, monthly. Each scheduled function runs in its own thread (pthreads on C backend, setInterval on JS runtime).

Wait Statement

Pause execution for a number of milliseconds:

func main() {
    print("starting...\n")
    wait 1000    // pause 1 second
    print("done\n")
}

10. Human-Centric Constructs

bc / because — Inline Explanation & Assertion

The bc keyword serves dual purpose: as an assertion (runtime check) and as an annotation (metadata for AI / debugging).

// Annotation (returns value unchanged, stores reason)
let age = 18 bc "minimum voting age"

// Assertion (terminates if condition is false)
bc age >= 16, "You must be at least 16 to drive."

// On function declarations
func discount(age: Int, purchases: [Float]) -> Float bc "Discount policy v3.2"

only — Guard / Exclusive Match / Ownership

// Guard in loops
for item in items {
    only item > 0 do
        process(item)
}

// Exclusive pattern match
match value {
    only Int   => print("exact integer")
    only Float => print("exact float")
    _          => print("other")
}

… (Ellipsis) — Range / Rest / Spread

// Range
for i in 1…5 { print(i) }   // 1,2,3,4,5

// Rest parameter
func average(…values: Float) -> Float { sum(values) / values.count }

// Spread into array
let more = [4, 5]
let all = [1, 2, 3, …more]   // [1,2,3,4,5]

// Destructuring with rest
let (first, …rest) = (1, 2, 3, 4)

11. AI & Documentation Standards

Every module, function, and type must have a /// Summary: comment — enforced by the compiler for production code.

/// Summary: Greet the user by name.
/// Parameters: name - the person to greet
/// Returns: a greeting string
func greet(name: String) -> String {
    "Hello, " + name + "!"
}

Recommended doc comment schema:

/// Summary: <one-line description>
/// Parameters: <param> - <description>, ...
/// Returns: <description>

12. Modules & Imports

// Basic import (items are inlined)
import "./math.ely"

// Aliased import (desugars alias.fn() to alias_fn())
    import "./math.ely" as math

    let a = 5
    let sq = math.square(a)
    let db = math.double(a)

Import paths are resolved relative to the importing file. Three prefix styles are supported:

  • ./ — relative path from the current file
  • @/ — project root (core/)
  • #/ — packages/ directory

13. Extensions

Extensions are optional language features that can add new keywords and C runtime functions to the compiler. Extensions are declared in elysium.json under the "extensions" field.

// elysium.json
{
    "name": "my-project",
    "extensions": ["extensions/json"]
}

Each extension directory contains an .ely file with an extension statement that declares the new keywords and their runtime implementations:

// extensions/json/json.ely
extension "json" {
    keywords: [
        "json.parse",
        "json.stringify",
        "json.buildObject"
    ],
    runtime: { other: "./runtime/json.c" }
}

The keywords array declares method prefixes that the compiler can desugar (e.g., json.parse(x) becomes __json_parse(x)). The runtime block maps platforms to implementation files — these are compiled and linked alongside the core runtime.

Extensions are automatically loaded when the compiler builds a project — no explicit imports are needed.

14. Packages

Packages are reusable modules written entirely in Elysium. They live in the packages/ directory as self-contained folders with their own elysium.json manifest.

Package Directory Structure

packages/@elysium/auth/
    elysium.json    # package manifest
    auth.ely        # source entry point
    LICENSE         # MIT license

Package Manifest (elysium.json)

{
    "name": "auth",
    "version": "0.1.0",
    "description": "Authentication library for Elysium",
    "entry": "auth.ely",
    "license": "MIT",
    "author": "Elysium Team",
    "repository": "https://github.com/imstevetran/ellysium-pl",
    "dependencies": {}
}

Importing a Package

Use the #/ prefix to import packages:

import "#/@elysium/auth" as auth

let token = auth.jwtSign("user123", "secret")

The compiler resolves #/name by looking up packages/name/elysium.json and reading its entry field.

Package CLI Commands

elysium init — Scaffold a New Package

Creates a new package directory under packages/ with a manifest and MIT license:

elysium init my-pkg --description "A useful library" --author "Me"

elysium install — Install Dependencies

Downloads packages from the EPM registry into elysium_modules/ and generates elysium.lock:

elysium install           # install all deps from elysium.json
elysium install auth      # install a specific package
elysium install auth@0.2.0  # install a specific version

elysium publish — Publish a Package

Validates the package manifest and prepares it for publishing to the EPM registry:

elysium publish                  # publish current directory's package
elysium publish ./packages/auth   # publish a specific package

elysium update — Check for Dependency Updates

Queries the EPM registry for newer versions of declared dependencies (use --apply to write updates to elysium.json):

elysium update                    # check all deps
elysium update auth --apply       # check and update auth

Lockfile

The elysium.lock file is generated automatically by elysium install and records exact versions of installed packages:

{
    "version": 1,
    "packages": {
        "auth": { "version": "0.1.0", "integrity": "sha256-..." }
    }
}

Full Example: Discount Calculator

This example showcases many Elysium features in one function: if … then, bc annotations, only guards, … spread, and enforced doc comments.

discount.ely
/// Summary: Calculate discount for a customer.
func discount(age: Int, purchases: [Float]) -> Float bc "Discount policy v3.2" {
    let adultAge = 18 bc "legal adult age"

    only age > 0 do {
        let base = if age >= adultAge then 0.10 else 0.05
        let total = sum(…purchases) bc "total of all purchases"
        let extra = if total > 500 then 0.05 else 0.0
        min(base + extra, 0.20)
    }
}
