Declarative UI Layer

Build reactive, component-based user interfaces with Elysium's built-in UI framework. No external libraries needed.

Two file formats: Use .elyx for XML-syntax components (TSX-style render {} block) and .ely for plain Elysium view syntax (function-call style). Both produce the same component semantics — it's purely a surface syntax choice.

What is .elyx?

.elyx is a TSX-like file format for Elysium UI components. An .elyx file contains exactly one component declaration with an optional render { … } block that uses XML syntax for the view tree.

Basic Structure

/// Summary: Counter component.
component Counter {
    state count = 0

    render {
        <Column>
            <Text>Count: {count}</Text>
            <Button label="Increment" onClick={count = count + 1} />
        </Column>
    }
}

How It Works

The render { … } block is desugared at compile time into standard Elysium view-tree construction calls. The component body (state declarations, methods, properties) is regular Elysium code — only the render block uses XML syntax.

.elyx vs .ely Component Syntax

The same component written in both formats:

// .elyx — XML render block
component Counter {
    state count = 0
    render {
        <Column>
            <Text>Count: {count}</Text>
            <Button label="Increment" onClick={count = count + 1} />
        </Column>
    }
}
// .ely — function-call view syntax
component Counter {
    state count = 0

    Column {
        Text("Count: ", count)
        Button(label: "Increment") onClick {
            count = count + 1
        }
    }
}

Core Concepts

  1. Component = Function that returns a View — Components describe a tree of views. They are automatically reactive.
  2. State — Declare mutable, observable variables with state. Any view reading a state variable re-renders when it changes.
  3. Views are immutable descriptions — No direct DOM manipulation. The runtime diffs and updates the actual UI.
  4. Event handlers — Attached as closures to views.
Philosophy: The UI layer is part of the language's standard library — not a framework bolted on. First-class syntax for views, state, and events with zero boilerplate.

Component Syntax

component Name {
    state varName = initialValue

    // View tree
    ViewType { ... }
}

State

Use state to declare mutable, observable variables. Views that read a state variable automatically re-render when it changes.

/// Summary: Counter with increment button.
component Counter {
    state count = 0 bc "current counter value"

    Column {
        Text("Count: ", count)
        Button(label: "Increment") onClick {
            count = count + 1
        }
    }
}

Conditional Rendering

Use if … then directly inside the view tree.

component Greeting {
    state isLoggedIn = true

    Column {
        if isLoggedIn then
            Text("Welcome back!")
        else
            Button(label: "Login") {
                isLoggedIn = true
            }
    }
}

Guards with only

component SecureContent {
    state userRole = "admin"

    only userRole == "admin" do {
        Text("Top secret data")
    }
}

Lists & Spread

Use the … ellipsis operator to spread lists of views.

/// Summary: Display a list of items.
component ItemList(items: [String]) {
    Column {
        …items.map(item =>
            Row {
                Checkbox(checked: false)
                Text(item)
            }
        )
    }
}

Two-Way Binding

Use = for binding between a UI input and state variable. Changes to the input automatically update the state.

component NameEditor {
    state name = "Alice"

    Column {
        Text("Your name:")
        TextField(value = name)
        Text("Hello, ", name)
    }
}

Event Handlers

Attach closures to view events via attributes like onClick. Inside a handler you can mutate state, call functions, or use bc assertions for validation.

The bc keyword is a general-purpose assertion — it works anywhere, not just in handlers. See the Language Guide for details.

component SubmitButton {
    Button(label: "Submit") onClick {
        bc isValidForm(), "Form must be valid before submitting"
        submitData()
    }
}

Styling

Views accept an optional style attribute — a plain dictionary of style properties.

Text("Important") style {
    color: "red",
    fontSize: 24,
    bold: true
}

Full Example: Todo App

A complete, production-style todo application combining all UI features. Shown here in both .elyx (XML) and .ely (function-call) syntax.

.elyx — XML render block

TodoApp.elyx
/// Summary: A simple todo list application.
component TodoApp {
    state tasks = [String]()
    state newTask = ""

    render {
        <Column padding="20">
            <Text style={{ fontSize: 24, bold: true }}>My Todos</Text>

            <Row>
                <TextField value={newTask} placeholder="Add a task" />
                <Button label="Add" onClick={ only newTask.trim() != ""; tasks = tasks + [newTask]; newTask = "" } />
            </Row>

            if tasks.isEmpty then
                <Text style={{ color: "gray" }}>No tasks yet!</Text>
            else
                <Column>
                    …{tasks.map((task, index) =>
                        <Row>
                            <Text>{task}</Text>
                            <Button label="Delete" onClick={ tasks = tasks.filter((_, i) => i != index) } />
                        </Row>
                    )}
                </Column>
        </Column>
    }
}

.ely — function-call view syntax

/// Summary: A simple todo list application.
component TodoApp {
    state tasks = [String]()
    state newTask = ""

    Column padding: 20 {
        Text("My Todos") style { fontSize: 24, bold: true }

        Row {
            TextField(value = newTask) placeholder "Add a task"
            Button(label: "Add") onClick {
                only newTask.trim() != "", "Task cannot be empty"
                tasks = tasks + [newTask]
                newTask = ""
            }
        }

        if tasks.isEmpty then
            Text("No tasks yet!") style { color: "gray" }
        else
            Column {
                …tasks.map((task, index) =>
                    Row {
                        Text(task)
                        Button(label: "Delete") onClick {
                            tasks = tasks.filter((_, i) => i != index)
                        }
                    }
                )
            }
    }
}
