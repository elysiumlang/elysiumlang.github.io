Spec-Driven Development

Write tests, benchmarks, and documentation inline with your code. Specs are first-class language constructs — not a separate framework.

Keywords Overview

KeywordAliasDescription
specdescribeDefine a test suite with a description
featitDefine an individual test case
expect—Assert that an expression is truthy
todo—Mark a test or section as not yet implemented
question—Flag an open concern or design question
benchbmMeasure wall-clock time of a block

Basic Spec Example

/// Summary: Example demonstrating spec-driven development.
func add(a: Int, b: Int) -> Int {
    a + b
}

spec "Calculator" {
    feat "adds two numbers" {
        let result = add(2, 3)
        expect result == 5
    }
    feat "adds negative numbers" {
        let result = add(-1, -2)
        expect result == -3
    }
}

describe / it Synonyms

describe is a synonym for spec, and it is a synonym for feat. Use whichever reads better for your context.

describe "Greeter" {
    it "says hello" {
        let msg = greet("World")
        expect msg == "Hello, World"
    }
}

todo & question Markers

todo and question are zero-runtime-cost annotations. They compile to nil and serve as documentation for unfinished or uncertain code.

spec "Calculator" {
    feat "adds two numbers" {
        let result = add(2, 3)
        expect result == 5
        todo "test negative numbers"
    }
    feat "division" {
        todo
        question "what about division by zero?"
    }
}

You can also use todo and question inside regular function bodies:

func main() {
    let x = 42
    question
    question "why is this here?"
    todo
    todo "fix the bug"
}

Benchmarking

Use bench / bm blocks to measure wall-clock runtime of any computation. The compiler emits clock_gettime calls around the block body.

func fibonacci(n: Int) -> Int {
    if n < 2 {
        n
    } else {
        fibonacci(n-1) + fibonacci(n-2)
    }
}

func main() {
    bm {
        let _ = fibonacci(30)
    }
}

Output format: bench: 0.012345 s printed to stdout.

Specs with Imports

Combine the import system with spec-driven development for cleanly organized test suites.

/// Summary: Testing with import-as and specs combined.
import "./math.ely" as math

spec "Math" {
    feat "square works" {
        let result = math.square(4)
        expect result == 16
    }
    feat "double works" {
        let result = math.double(3)
        expect result == 6
    }
}

Complete Keyword Reference

A comprehensive example using every spec keyword.

// All spec-driven keywords

spec "Math Tests" {
    feat "addition" {
        expect 1 + 1
        expect 2 + 2
    }
    feat "subtraction" {
        expect 5 - 3
    }
}

describe "Other Tests" {
    it "boolean" {
        expect true
        expect !false
    }
    it "todo markers" {
        todo
        todo "implement this"
    }
}

func main() {
    bench {
        // measure something
    }
    bm {
        // shorthand benchmark
    }
}
