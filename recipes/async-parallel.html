Async & Parallel Execution

Concurrent and parallel execution primitives. Intermediate

Parallel Blocks

Each statement inside a parallel block runs in its own thread. In the C backend this uses pthread_create/pthread_join for true parallelism. In the JS runtime it uses Promise.all.

func main() {
    parallel {
        print("task a\n")
        print("task b\n")
        print("task c\n")
    }
    print("all parallel tasks done\n")
}

// Multiple parallel blocks can be used
func processBatch() {
    parallel {
        print("batch item 1\n")
        print("batch item 2\n")
    }
}

Async / Await

Async functions declare that they execute asynchronous work. await is recognized only inside async context.

/// Async function declaration
async func fetchData() -> string {
    let data = await httpGet("https://api.example.com")
    return data
}

// Lightweight tasks (green threads)
async { producer(ch) }
async { consumer(ch) }

Channels & Select

let ch = Channel<Int>(capacity: 3)

async func producer(ch: Channel<Int>) {
    for i in 1..5 {
        await ch.send(i)
    }
    ch.close()
}

async func consumer(ch: Channel<Int>) {
    while let value = await ch.receive() {
        print("Got: ", value)
    }
}

// Select on multiple channels
select {
    case msg = ch1 => handle(msg)
    case msg = ch2 => handle(msg)
}

Implementation Notes

  • Parallel blocks: v1 generates pthread-based parallelism. Each statement's code is wrapped in a void* __parallel_wrapper_N(void* arg) function.
  • Await: v1 C backend executes awaited expressions synchronously inline. A full state-machine transform (heap-allocated frame + state variable switch) is planned for v2.
  • JS Runtime: __await wraps values in Promise.resolve, __async ensures functions return Promise, __parallel uses Promise.all.
