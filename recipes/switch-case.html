Switch / Case Branching

Human-friendly branching with switch/case/then/else. Beginner

Switch as Expression

switch_expression.ely
func describe_score(score: Int) -> String {
    return switch score {
        case 100 then { "Perfect!" }
        case 90 then { "Excellent!" }
        case 80 then { "Great!" }
        case 70 then { "Good" }
        case 60 then { "Passing" }
        else { "Needs improvement" }
    }
}

let s1 = describe_score(100)   // "Perfect!"

Switch as Statement

let x = 3
switch x {
    case 1 then { print("one\n") }
    case 2 then { print("two\n") }
    case 3 then { print("three\n") }
    else { print("many\n") }
}

Wildcard Pattern

let y = 42
switch y {
    case 0 then { print("zero\n") }
    case _ then { print("non-zero\n") }
}

String Patterns

let name = "bob"
switch name {
    case "alice" then { print("hi alice\n") }
    case "bob" then { print("hi bob\n") }
    else { print("hi stranger\n") }
}

How It Works

switch/case/then/else desugars directly into match/case AST nodes at compile time. The entire pipeline (HIR, MIR, codegen) sees them as standard pattern matching — no special handling needed. else becomes a Wildcard pattern arm.
