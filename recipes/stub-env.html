Stub Functions & Environment Filtering

Declare interfaces without implementations and filter them by environment. Advanced

Stub Basics

Stub functions declare a function signature without a body. They compile to a function with an empty body and a default return value.

/// Available in all environments
func placeholder(x: Int) -> Int stub

/// Only in local and dev
func debugOnly(key: String) -> String stub: [local, dev]

/// Only in test
func testHelper() -> String stub: [test]

Normal vs Stub

/// A real implementation
func normalFunc() -> Int {
    return 42
}

/// Cross-env stub (matches all environments)
func crossEnv() -> Float stub

How Environment Filtering Works

  1. Parser produces stub_envs: Option<Vec<String>> on the function AST node.
  2. Bare stub (no env list) → matches all environments.
  3. stub: [local, test] → matches only local and test environments.
  4. Before type-checking/codegen, filter_stubs() removes functions whose env list doesn't match --env.
  5. The type checker, ownership checker, and linter skip body-walking for stub functions.
  6. Stub functions get an empty body Block { statements: [] } — just a default return.

CLI Usage

elysium build app.ely ---env test
elysium run app.ely ---env dev
elysium check app.ely ---env prod

Supported built-in environments: local (default), dev, test, prod. Custom aliases can be defined in elysium.json.
