String Utilities & Cryptography

30+ string methods available as namespace calls or method-call syntax. Intermediate

Basic String Operations

func main() {
    // Length and inspection
    let len1 = string.length("hello")
    let len2 = "hello".length()        // method-call syntax works too
    let isEmpty = "".isEmpty()
    let hasHello = "hello world".contains("hello")
    let starts = "hello".startsWith("he")

    // Transformation
    let upper = "hello".toUpper()
    let lower = "HELLO".toLower()
    let trimmed = "  hi  ".trim()
    let replaced = "hello world".replace("world", "there")
    let sliced = "hello".slice(1, 4)
    let parts = "a,b,c".split(",")
}

Cryptographic Operations

func main() {
    // Hashing
    let hash = "hello".sha256()
    let md = "hello".md5()

    // Encoding
    let b64 = "hello".base64Encode()
    let decoded = "aGVsbG8=".base64Decode()
    let hex = "hello".hexEncode()

    // HMAC
    let signature = "message".hmac("secret")

    print("crypto operations completed\n")
}

Method-Call vs. Namespace Syntax

Both syntaxes compile to the same __string_* builtin calls:

NamespaceMethod-Call
string.length("hello")"hello".length()
string.sha256("hello")"hello".sha256()
string.base64Encode("hello")"hello".base64Encode()

The method-call syntax desugars by cloning the receiver and prepending it as the first argument. Use whichever is more readable.

Backend Details

  • C backend: Uses strlen, strstr, strncmp, snprintf, and GEP for most operations. Crypto uses popen with openssl/xxd.
  • JS runtime: Maps to native String.prototype methods. Crypto uses Node's crypto.createHash/createHmac and Buffer.
